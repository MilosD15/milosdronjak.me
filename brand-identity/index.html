Brand identity and styling rules

Use rule IDs below when requesting changes (example: "update BR-03 to 1rem"). Each rule includes what to use and where it currently applies.

BR-01

Color tokens only

Always use token colors from `:root` in `css/styles.css`.

  • Primary text: `--ink`
  • Main accent: `--orange`, `--orange-dark`, `--orange-darker`
  • Soft section background: `--blush`
Ink Orange Orange dark Blush bg

BR-02

Typography hierarchy

Headings use Fraunces. Body/UI text uses Montserrat.

  • Section headings: `u-type-h2`
  • Body copy: `u-type-body` or tokenized component text
  • Emphasis: semibold lead + muted body

BR-03

Body text scale

Body copy is consistent across pages.

  • Desktop: `0.92rem`
  • Mobile: `0.82rem`
  • Prefer tokens over hardcoded px

Desktop body sample (0.92rem).

BR-04

Container alignment

All section content should align to the same shell.

  • Use `max-width: var(--max)`
  • Horizontal gutter: `padding: 0 var(--space)`
  • Use `box-sizing: border-box; width: 100%` for inner wrappers

BR-05

Buttons and CTA language

Reuse shared button styles (`btn` modifiers) and keep CTA copy concise.

  • Primary CTA: `btn btn--primary`
  • Secondary CTA: `btn btn--ghost`
  • Avoid one-off button CSS when a shared modifier exists

BR-06

Animation style

Use soft fade/translate transitions and section reveal choreography.

  • Reveal class pattern: `.reveal`, `.reveal--up`, `.is-revealed`
  • Copy switches: `is-updating` + `copyFade` timing
  • Respect reduced motion in CSS and JS

BR-07

Section content pattern

When a section needs explanation text, use lead/sub text structure.

  • Lead line: semibold (`strong`)
  • Supporting paragraph: muted text (`span`/body)
  • Used in homepage Description and tools-related sections

Every Shopify agency has the same quiet backlog. Supporting sentence follows in muted tone.

BR-08

Breakpoints

Use the two shared breakpoints unless there is a specific reason.

  • Tablet: `960px`
  • Mobile: `640px`
  • Mobile nav behavior follows `ui.breakpoints.navMobile` from `MilosSite`
